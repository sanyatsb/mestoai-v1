# Gonka Bot

Telegram AI bot powered by **Kimi K2.6** on the **Gonka** decentralized network. Free for end users, no signup. International audience, 6 UI languages.

**Status:** Week 1 / 7-8 (Foundation skeleton). See [Gonka_Bot_MVP_TZ_v1.2.md](Gonka_Bot_MVP_TZ_v1.2.md) for the full spec and [BACKLOG.md](BACKLOG.md) for Phase 2+.

---

## Stack

TypeScript 5.4 strict + Node 22 LTS + pnpm 9 · grammY (bot framework) · Drizzle ORM + Postgres 16 · Redis 7 · Hono (HTTP) · pino (logs) · vitest (tests) · Biome (lint+format) · Docker Compose + Caddy (HTTPS).

## Prerequisites

- Node 22+ (`nvm install 22 && nvm use 22`)
- pnpm 9+ (`corepack enable && corepack prepare pnpm@latest --activate`)
- Docker Desktop (for Postgres / Redis / production deploy)
- ffmpeg — only required if running the bot *outside* Docker (it's baked into the image). `brew install ffmpeg` on macOS.

## Quick start (local dev)

```bash
# 1. Install deps
pnpm install

# 2. Provision config
cp .env.example .env
# Fill in BOT_TOKEN, BOT_USERNAME, GONKA_API_KEY, OPENAI_API_KEY,
# WEBHOOK_URL (https://...), WEBHOOK_SECRET (openssl rand -hex 32),
# ADMIN_TG_IDS, ADMIN_CHAT_ID.

# 3. Bring up Postgres + Redis
docker compose up -d postgres redis

# 4. Apply migrations
pnpm db:migrate

# 5. Run the bot (uses long-polling in development — no ngrok needed)
pnpm dev

# 5b. Or with pretty logs:
pnpm dev:pretty

# 6. In another terminal, smoke test
curl http://localhost:8080/health     # {"status":"ok"}
curl http://localhost:8080/ready      # {"status":"ok","db":true,"redis":true}
# Send /start to your bot in Telegram → expect a "Hello, world! 👋" reply.
```

## Scripts

| Script | Purpose |
|--------|---------|
| `pnpm dev` | Hot-reload dev server (tsx watch) |
| `pnpm dev:pretty` | Same, piped through `pino-pretty` |
| `pnpm build` | Compile to `dist/` |
| `pnpm start` | Run compiled `dist/main.js` |
| `pnpm test` | Run vitest once |
| `pnpm test:watch` | Watch mode |
| `pnpm lint` | Biome check |
| `pnpm lint:fix` | Biome autofix |
| `pnpm db:generate` | Generate a new migration from `src/db/schema.ts` |
| `pnpm db:migrate` | Apply pending migrations |
| `pnpm db:studio` | Open Drizzle Studio (web UI) |
| `pnpm set-webhook` | Register the Telegram webhook with secret token |

## Migration notes

After `pnpm db:generate`, **manually** add the partial unique index (Drizzle can't express it yet) — see [AUDIT-M11, N4] in `src/db/schema.ts`. Create `drizzle/migrations/9999_partial_index.sql` containing:

```sql
CREATE UNIQUE INDEX IF NOT EXISTS uniq_user_active_conv
  ON conversations(user_id)
  WHERE is_active = true AND group_chat_id IS NULL;
```

Then `pnpm db:migrate` applies it alongside the autogenerated schema.

## Bumping `CURRENT_TOS_VERSION`

[AUDIT-H10, B2] Bumping `CURRENT_TOS_VERSION` in `.env` (e.g. `1.0` → `1.1`) makes the auth middleware reset `tos_accepted_at` to NULL on the next `/start` from each existing user, forcing a re-accept of the new terms. Make sure the updated ToS text is present in `src/locales/*.json` *before* you bump the version — otherwise users see stale wording.

## Persona reload

[AUDIT-L18] Persona definitions live in `src/data/personas.yaml`. On startup `scripts/seed-personas.ts` (Week 3) **upserts** all entries by `slug`. Editing the YAML at runtime won't take effect until the next deploy / container restart.

## Deployment

See §13 of [Gonka_Bot_MVP_TZ_v1.2.md](Gonka_Bot_MVP_TZ_v1.2.md). The short version:

```bash
# On a fresh Ubuntu 24.04 VPS with Docker installed:
git clone <repo> && cd gonka-bot
cp .env.example .env && nano .env

docker compose up -d
# Container CMD runs `migrate.js && main.js` automatically.

docker compose exec bot node dist/scripts/set-webhook.js
curl https://bot.yourdomain.com/health
curl https://bot.yourdomain.com/ready
```

For the host-side `pg_dump` cron, see §13.5 of the TZ.
